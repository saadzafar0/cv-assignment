# RearGuard

A bare React Native Android app that turns a phone mounted facing backward in
a car into a real-time rear-collision warning system. The rear camera streams
frames through an on-device MobileNet-SSD COCO TFLite model; the bounding-box
pixel width of the nearest detected vehicle is converted into a real-world
distance via the pinhole-camera formula and mapped to coloured alert zones.

| Zone    | Distance      | UI                | Audio          | Haptics      |
| ------- | ------------- | ----------------- | -------------- | ------------ |
| Safe    | > 5 m         | green zone bar    | silent         | none         |
| Caution | 2 m – 5 m     | yellow zone bar   | 1 Hz beep      | none         |
| Danger  | < 2 m         | red zone bar + pulsing red border | ~3 Hz beep | 200 ms vibration per beep |

## Tech stack

- React Native 0.85 (bare workflow)
- TypeScript only
- `react-native-vision-camera` v4 + `react-native-worklets-core` for camera & frame processors
- `vision-camera-resize-plugin` to convert camera frames to a 300x300 RGB tensor
- `react-native-fast-tflite` running the `coco_ssd_mobilenet_v1_1.0_quant_2018_06_29` model
- `react-native-reanimated` v3 for the danger pulse + UI animations
- `react-native-sound` for the beep, played from a procedurally-generated WAV
- `@react-native-async-storage/async-storage` to persist the calibrated focal length
- `@react-navigation/native-stack` for the Main / Calibration screens

Android only. iOS targets exist in the workspace but aren't part of the assignment scope.

## Algorithm

```
focalLength    = (pixelWidth_at_known_distance * known_distance) / real_object_width
distance_cm    = (REAL_CAR_WIDTH_CM * focalLength) / pixelWidth
```

Defaults: `REAL_CAR_WIDTH_CM = 170`, `DEFAULT_FOCAL_LENGTH = 850`. The
calibration wizard records `pixelWidth` of a real car parked exactly 200 cm
behind the phone and persists the resulting `focalLength` in AsyncStorage.

The displayed distance is a rolling average of the last 3 valid frames, which
suppresses jitter from the SSD detector without introducing noticeable lag.

## Getting started

> Prerequisites: Node ≥ 22, JDK 17, Android SDK + a real Android device
> (the camera + ML pipeline is too slow on emulators for a useful demo).

```sh
npm install                # also runs scripts/fetch-assets.js as postinstall
npm run setup:assets       # re-run any time you delete the model / beep
npm start                  # Metro
npm run android            # build + install on connected device
```

The first time the app launches it will request CAMERA + RECORD_AUDIO
permissions (the latter is required by vision-camera v4 even when not
recording audio). Tap **CALIBRATE** in the bottom-left to run the wizard the
first time you use a different phone or mount; the default focal length of
850 px works as a fallback if you skip calibration.

## Notes & caveats

- The MobileNet-SSD COCO model only knows about `car`, `truck`, `bus`, and
  `person` from this app's perspective; everything else is filtered out.
- Distance accuracy is a function of how well calibrated the focal length is
  *and* how close the real-world object width matches `REAL_CAR_WIDTH_CM`. A
  truck or van will read closer than it really is unless you recalibrate.
- The frame processor runs at 15 fps to keep latency low without burning
  battery. Bump `FRAME_PROCESSOR_FPS` in `src/utils/constants.ts` if you
  want smoother box overlays.
